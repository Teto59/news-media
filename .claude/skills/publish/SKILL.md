# /publish — 差分学習 + 公開ワークフロー

**引数**: slug(`/article` で作成した記事のslug。ファイル名は `YYYY-MM-DD-<slug>.md`)

## 手順

1. **初稿との差分を取得する**
   - `git log --oneline -- <記事ファイル>` で `draft: <slug>` コミットを特定する
   - `git diff <draftコミット> -- <記事ファイル>` に、未コミットの変更があればそれも合わせて、**初稿 ↔ オーナー最終稿の差分**を取得する

2. **教訓を抽出し `editorial/LESSONS.md` に追記する**
   - フォーマットを厳守する:
     ```
     - [YYYY-MM-DD][分類: タイトル/文体/構成/事実確認/その他] 一般化した教訓1行(具体例: 「X」→「Y」)
     ```
   - 日付は今日の日付。分類は差分の内容に応じて選ぶ(複数種の修正があれば教訓を複数行に分けてよい)
   - 教訓は**具体的すぎず一般化する**こと。次の `/article` 実行で活かせる形にする
   - **差分がゼロの場合も**「今回の初稿で通用した点」を1行記録する(成功パターンの強化。分類は「その他」でよい)

3. **公開用に記事を確定する**
   - frontmatterの `draft: false` にする
   - `pubDate` を公開日(オーナー指示があればそれに従う。既定は今日)に確定する
   - `node scripts/gen-og.mjs <slug>` でOG画像を生成する
   - `npm run build` がエラー0で通ることを確認する

4. **公開コミットとpush**
   - 記事・OG画像・LESSONS.mdの変更を `git add` し、コミットメッセージ `publish: <slug>` でコミットする
   - **push は他者から見える公開行為であり、実行前に必ずオーナーに最終確認を取ってから行う**(ドライラン時はオーナーの指示に従い push を省略する)
   - push後、数分待ってから本番URL(`https://<プロジェクト名>.pages.dev/articles/<日付>-<slug>/`)で表示を確認し、オーナーに結果を報告する

## 注意

- LESSONS.mdへの追記は「オーナーの添削を否定・評価する」ためではなく「次の初稿の質を上げる」ために行う。差分の背景にある理由まで踏まえて一般化すること
