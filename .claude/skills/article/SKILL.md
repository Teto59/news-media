# /article — 記事執筆ワークフロー

**引数**: ニュースのURL、または取材したいテーマ(文字列)

## 手順

1. **前提知識を読み込む**
   - `editorial/STYLE_GUIDE.md` と `editorial/LESSONS.md` を全文読む。
   - `src/content/articles/` の記事を `pubDate` 降順で新しい順に3本読み、直近の文体・構成の感覚を掴む。

2. **独立ソースを調査する**(WebSearch / WebFetch を使用)
   - 独立した情報源を**3件以上**調査し、うち**2件以上**を実際に出典として使えることを確認する(同一発表の転載は1つと数える)。
   - 公式発表・決算資料・論文・当事者発言などの**一次ソースを最優先**する。
   - **paywallで本文を確認できないものは出典にしない**。
   - 出典を確認できない事実主張は書かない(`STYLE_GUIDE.md` 第1条)。裏取りできない情報は「不明」と書くか、記事にしない。

3. **記事を執筆する**(`src/content.config.ts` のzodスキーマに完全準拠。`draft: true` で作成)
   - ファイルパス: `src/content/articles/YYYY-MM-DD-english-slug.md`(`YYYY-MM-DD` は今日の日付、slugは内容が分かる短い英語)
   - frontmatter:
     - `title`: 48字以内。煽り疑問形・扇動語・釣り見出し禁止(STYLE_GUIDE準拠)
     - `lead`: 80〜180字
     - `tldr`: 3項目、各60字以内
     - `pubDate`: 今日の日付
     - `tags`: 1〜4個。**第一タグは記事のジャンル**(政策・科学・政治・経済・テック 等)
     - `sources`: 各項目に `name` / `url` / `publisher`(任意) / `accessed`(参照日)。2件以上
     - `corrections`: 初稿では空配列(`[]`)
     - `draft: true`
   - 本文:
     - 事実主張には出典番号 `[1]` 形式を付ける
     - 見立て・解釈は本文と分離し、`## 分析` 見出しの節にまとめ、「〜とみられる」等の推量表現を使う
     - です・ます調、一文60字程度を目安、カタカナ語の乱用を避け初出の専門用語は一言で補足(STYLE_GUIDE準拠)

4. **サムネ生成とビルド確認**
   - `node scripts/gen-thumb.mjs <slug>` を実行してサムネSVGを生成する
   - `npm run build` がエラー0で通ることを確認する

5. **初稿としてコミットする**
   - 記事ファイルとサムネ(`public/thumbs/<slug>.svg`)を `git add` し、コミットメッセージ `draft: <slug>` でコミットする
   - **このコミットが「初稿」であり、`/publish` 実行時の差分学習の基準点になる。ここで push はしない。**

6. **オーナーに報告する**
   - `npm run dev` でのプレビュー方法を案内する
   - レビュー観点を提示する: タイトルは正確か / リードは読みたくなるか / 事実誤認はないか
   - 添削が終わったら `/publish <slug>` を実行してもらう旨を伝える

## 禁止事項

- 出典のない事実主張を書くこと
- 独立ソース1件のみで執筆すること
- 有料記事1本だけを情報源にすること(著作権ルール: 特定記事の表現・構成の引き写しも禁止)
- `src/content.config.ts` のスキーマから逸脱すること
