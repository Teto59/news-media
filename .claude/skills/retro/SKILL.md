# /retro — 教訓の整理・昇格ワークフロー

**引数**: なし(オーナーが月1回目安で起動する)

## 手順

1. `editorial/LESSONS.md` を全文読む。
2. 内容が近い教訓をグルーピングし、**3回以上繰り返されている教訓**を特定する。
3. 3回以上繰り返された教訓を一般化したルール文にして、正式ルールとして追記する:
   - 文体・タイトル・構成・出典・事実確認に関する教訓 → `editorial/STYLE_GUIDE.md` の該当節
   - デザインに関する教訓 → `editorial/DESIGN_NOTES.md`(ただしパレット・フォント・radiusの変更は不可。変更する場合は理由を追記してから行う)
4. 昇格させた教訓は `editorial/LESSONS.md` から削除する(3回未満のものはそのまま残す)。
5. 内容が矛盾する教訓が見つかった場合は、昇格せずにオーナーへ日本語で平易に質問して解消する(オーナーは非エンジニア。専門用語は毎回一言で補足する)。
6. 整理結果(昇格したルール・削除した教訓・保留した論点)をオーナーに簡潔に報告する。
