# DESIGN_NOTES.md — デザイン仕様の写し + デザイン変更の教訓

`docs/PLAN.md` に記載のデザイン仕様の写し。デザインを変更する場合は理由をこのファイルに追記してから行う(`CLAUDE.md` 参照)。パレット・フォント・radius は固定であり、独自判断で変更しない。

## コンセプト

「紙の新聞の信頼感 × 現代の静的サイト」。生成り紙×墨×藍×朱の和のパレット。鋭角(radius 2px)、罫線主体、影なしのフラットな紙面。

## デザイントークン(`src/styles/global.css`)

全コンポーネントはトークン経由でのみ色を使う(hex直書き禁止)。

```css
:root {
  --paper:   #F6F1E7;  /* 背景: 生成り紙 */
  --surface: #FDFBF5;  /* カード面 */
  --ink:     #211D16;  /* 本文: 墨 */
  --ink-2:   #6B6353;  /* 補助テキスト */
  --line:    #DDD3BE;  /* 罫線 */
  --ai:      #2A5A8C;  /* 藍: リンク・主アクセント・タグ */
  --shu:     #C2451E;  /* 朱: 訂正・強調・「NEW」 */
  --radius: 2px;
}
[data-theme="dark"] {
  --paper: #191611; --surface: #211D16; --ink: #EBE4D3;
  --ink-2: #A39A85; --line: #3A3428; --ai: #8FB4DC; --shu: #E27452;
}
```

## タイポグラフィ

自己ホスト(@fontsource)。

- 見出し・サイトロゴ = Shippori Mincho B1(700/800)
- 本文・UI = Zen Kaku Gothic New(400/500)
- 日付・出典番号・英数 = IBM Plex Mono
- 本文16px・行間1.9・記事本文は `max-width: 38em`
- `font-feature-settings: "palt"` を見出しに適用

## トップページ

上部に Masthead(明朝の大きなロゴタイプ+今日の日付をIBM Plex Monoで+テーマ切替+検索リンク)。最新1本を横長の大型カード、以下3列グリッド(タブレット2列・スマホ1列)のカード。カード構成 = サムネSVG(16:9)→タグチップ→タイトル(明朝)→リード2行(`-webkit-line-clamp: 2`)→日付。

Phase 2で最新1本の大型カード(サムネ+本文を横並び)+3列グリッド(タブレット2列・スマホ1列)+staggered fade-inを実装済み。

トップは最新12本まで(大型カード1+グリッド11)に表示を制限し、13本以上ある場合のみグリッド下に「過去の記事一覧 →」リンク(`--ai`のテキストリンク、右寄せ、font-mono)を表示して `/archive` に誘導する(2026-07-02追加)。

## アーカイブページ(`/archive`、2026-07-02追加)

トップに表示しきれない過去記事の一覧。`getPublishedArticles()` の結果をpubDate降順のまま年月ごとにグルーピングし、見出しは「2026年7月」形式。各月の見出しはトップの `.section-head`(藍の角マーク■+明朝+下罫線)と同じ意匠。各記事は1行リストで「日付(font-mono、YYYY/MM/DD)+タイトル(ゴシック、リンク)+第一タグ(font-mono・`--ink-2`の小さめ表示)」を横並びにし、行間を細罫(`--line`)で区切る。ページ全体は記事ページと同じ `max-width: 44rem`、見出しは明朝のh1「アーカイブ」。Masthead の検索リンクの左に同スタイルの「一覧」リンクを追加して導線を作った。

## 記事ページ

タイトル(明朝 `clamp(1.6rem,4vw,2.4rem)`)→メタ行(日付・タグ・読了目安=本文字数÷500字/分)→ TldrBox(「3行まとめ」見出し、`border-left: 3px solid var(--ai)`)→本文→「分析」節(ある場合)→ SourceList(番号付き出典リスト。publisherと参照日を表示)→ CorrectionNote(correctionsがある場合のみ、`var(--shu)` の罫線で目立たせる)→ RelatedArticles(同一タグの最新3本)。

## モーション

ページロード時にカードを `animation-delay` で40msずつ順にfade-in + 8px上昇(CSSのみ)。カードhoverは `border-color: var(--ai)` + `translateY(-2px)`、`transition: all 160ms ease-out`。`prefers-reduced-motion` を尊重する。

## ダークモード

`prefers-color-scheme` 初期値 + ThemeToggle(選択をlocalStorageに保存。個人情報ではないのでOK)。FOUC防止のため `<head>` 内インラインスクリプトで `data-theme` を先に設定する。

## サムネSVG(`scripts/gen-thumb.mjs`)

slugの文字列ハッシュを乱数シードにした決定論的生成(同じ記事は常に同じ絵)。**ポスター調の大胆な色面構成**: 第一タグで決まる主色(藍/朱/深緑 #3F6B4F/黄土 #A67C2E。各色に濃色バリアントあり)の**ベタ地**に、紙色(#F6F1E7)と墨色の大きな幾何形(斜め帯/円環/対角分割のいずれか1構成をシードで選択)を高い不透明度で重ねる。文字は入れない。ダーク対応不要(画像として固定)。

※初版は「紙色の地+低opacityの幾何模様」だったが、画面の大半を占めるサムネがほぼ無地に見え、ページ全体が単調な淡色になったため上記に変更(2026-07-02)。「dominant colors with sharp accents」(frontend_aesthetics)に沿い、色の主役はサムネが担う。

### サムネの手動差し替え(2026-07-02追加)

frontmatterに `thumb: "/thumbs/xxx.jpg"` を指定すると、自動生成SVGの代わりにその画像を表示する(カード・記事・OG画像すべて)。ファイルは `public/thumbs/` に置く。**JPGまたはPNG推奨**(OG画像はSVG非対応のSNSが多いため)。

**権利上使える画像のみ**: 自分で撮影した写真/公式プレスキット配布素材/ライセンス上使用可の素材。他社報道の写真の転用は著作権侵害なので禁止(記事本文と同じルール)。

## OG画像(`scripts/gen-og.mjs`)

記事のサムネSVG(16:9・800x450)を、OGP標準比率(1200x630)にcoverフィット(拡大して中央クロップ)させてPNG化する。サムネと同様に文字は入れない。

## デザイン変更履歴

- Phase 2: favicon.svg をAstroデフォルトのロケットロゴから、トークン準拠のデザインに変更。サイト名が「(仮称)」で未確定のため、暫定的に明朝体1文字「報」(ink/paper トークン・ダーク対応)を仮のロゴマークとして採用。サイト名確定時に `src/consts.ts` の更新と合わせて差し替えること。
- 2026-07-02(Phase 2後の手直し): オーナーから「AIっぽい没個性デザインになっている」との指摘。原因はサムネの淡さ(ほぼ無地のベージュ矩形が画面を支配)と紙面らしい階層の欠如。パレット・フォント・radiusは一切変えず、以下を変更:
  1. サムネを上記のポスター調ベタ地構成に刷新(全記事再生成)
  2. Masthead に新聞の題字らしい二重罫(墨2px+細罫1px)と、favicon と揃えた朱の角印「報」を追加
  3. ページ最上部に3pxの藍帯(新聞のカラーバー)
  4. トップの記事グリッドに段見出し「記事一覧」(藍の角マーク付き)、最新記事カードに朱の「最新」ラベル
  5. タグチップを藍の色面(淡い藍地+hoverで藍地白抜き)に強化、カードの日付行に罫線
  6. カード日付行の上に罫線を入れ、紙面の「段」の区切りを明確化
- サイト名Chronica決定。角印・faviconを紀に変更(2026-07-02)
- 2026-07-02: トップの大型カード(最新記事)でサムネが見切れる問題を修正。従来はサムネの高さを本文側に合わせて `object-fit: cover` でクロップしていたため、16:9の手動サムネ画像の左右が欠けていた。大型カードのサムネを常に16:9のまま全体表示する方式に変更(クロップ廃止)。サムネと本文の間の縦罫はサムネの `border-right` から本文側の `border-left` に移し、本文がサムネより縦長になる場合はサムネを上下中央に置く
